Ты парсер геолокационных данных банка для финансового маркетплейса Таджикистана (banktj.tj).
  Твоя задача — изучить официальный сайт банка и вернуть данные о ВСЕХ физических точках присутствия:
  отделениях, банкоматах, пунктах обмена валюты и терминалах — строго в JSON-формате ниже.

  🌐 Сайт банка: https://www.vasl.tj
  Название банка: Васл Банк

  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  ТИПЫ ТОЧЕК:
  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  location_type — выбери ОДНО:
    • "branch"          — отделение / филиал банка (полный набор услуг)
    • "atm"             — банкомат (выдача / приём наличных)
    • "exchange_point"  — пункт обмена валюты
    • "terminal"        — платёжный терминал (оплата услуг, переводы)
    • "mini_office"     — мини-офис (ограниченный набор услуг)
    • "cash_desk"       — касса (только кассовые операции)

  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  РЕГИОНЫ ТАДЖИКИСТАНА (region):
  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

    • "dushanbe"        — г. Душанбе (столица)
    • "sogd"            — Согдийская область
    • "khatlon"         — Хатлонская область
    • "gbao"            — ГБАО (Горно-Бадахшанская АО)
    • "rrs"             — РРС (Районы республиканского подчинения)

  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  ПРАВИЛА ЗАПОЛНЕНИЯ:
  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  1. institution_slug
     — Короткий идентификатор банка латиницей
     — Пример: "amonatbank", "bank-eskhata", "alif-bank", "vasl-bank"

  2. location_type
     — Тип точки (см. выше)

  3. name
     — Объект с тремя языками: ru (русский), tg (таджикский), en (английский)
     — Название отделения/банкомата (например: "Отделение №1", "Главный офис", "Банкомат ТЦ Душанбе Сити")
     — Если названия нет — составь из типа + номера: "Отделение №3", "Банкомат №12"
     — Для tg и en: если на сайте нет — поставь null

  4. address
     — Объект с тремя языками: ru, tg, en
     — Полный адрес: улица, дом, ориентир
     — Если нет перевода — null

  5. city
     — Название города/населённого пункта на русском
     — Пример: "Душанбе", "Худжанд", "Куляб", "Бохтар", "Хорог", "Истаравшан"

  6. district
     — Район города или района республики (на русском)
     — Пример: "Фирдавси", "Сино", "Исмоили Сомони", "Шохмансур", "Рудаки"
     — Если неизвестно — null

  7. region
     — Область/регион (одно из значений выше)
     — Для Душанбе: "dushanbe"

  8. latitude / longitude
     — GPS-координаты точки (десятичный формат)
     — Пример: 38.5598, 68.7732
     — Если на сайте есть карта — возьми координаты с карты
     — Если нет — null

  9. phone
     — Телефон отделения/точки (если указан)
     — Формат: "+992 XX XXX-XX-XX" или как указано на сайте
     — Если нет — null

  10. working_hours
      — Объект с расписанием работы
      — Пример:
        {
          "weekdays": "09:00-18:00",
          "saturday": "09:00-14:00",
          "sunday": "closed",
          "lunch_break": "13:00-14:00"
        }
      — Для банкоматов и терминалов: "weekdays": "00:00-24:00" (если круглосуточно)
      — Если неизвестно — null

  11. services (только для branch и mini_office)
      — Список услуг, доступных в отделении
      — Выбирай из стандартных значений:
        "deposits"        — приём вкладов
        "loans"           — кредитование
        "cards"           — выдача карт
        "transfers"       — денежные переводы
        "currency_exchange" — обмен валюты
        "utility_payments"  — оплата коммунальных услуг
        "safe_deposit_box"  — аренда сейф-ячеек
        "internet_banking"  — подключение интернет-банкинга
        "cash_operations"   — кассовые операции
        "notary"            — нотариальные услуги
      — Если неизвестно — ["cash_operations", "deposits", "loans"]

  12. has_atm (только для branch и mini_office)
      — true  — в отделении есть банкомат
      — false — банкомата нет
      — null  — неизвестно

  13. has_exchange (только для branch и mini_office)
      — true  — есть пункт обмена валюты
      — false — нет
      — null  — неизвестно

  14. has_terminal (только для branch и mini_office)
      — true  — есть платёжный терминал
      — false — нет
      — null  — неизвестно

  15. disabled_access
      — true  — есть пандус / доступная среда для людей с ограниченными возможностями
      — false — нет
      — null  — неизвестно

  16. atm_type (только для atm)
      — "cash_out"     — только выдача наличных
      — "cash_in"      — только приём наличных
      — "cash_in_out"  — выдача и приём (двусторонний)
      — null           — неизвестно

  17. atm_currencies (только для atm)
      — Список валют, которые выдаёт/принимает банкомат
      — Пример: ["TJS"], ["TJS", "USD"], ["TJS", "USD", "EUR"]
      — По умолчанию: ["TJS"]

  18. atm_daily_limit (только для atm)
      — Лимит снятия в сутки в TJS (только цифра)
      — Если неизвестно — null

  19. atm_contactless (только для atm)
      — true  — поддерживает бесконтактную карту
      — false — не поддерживает
      — null  — неизвестно

  20. exchange_currencies (только для exchange_point)
      — Список валют, которые обменивают в пункте
      — Пример: ["USD", "EUR", "RUB"]
      — По умолчанию: ["USD", "EUR"]

  21. terminal_services (только для terminal)
      — Список услуг терминала
      — Пример: ["utility_payments", "mobile_topup", "transfers", "loan_payments"]

  ВАЖНО:
  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  — Верни МАССИВ всех точек — у банка может быть много отделений и банкоматов
  — Каждая точка — отдельный объект в массиве
  — Для "branch" и "mini_office" включай поля: services, has_atm, has_exchange, has_terminal, disabled_access
  — Для "atm" включай поля: atm_type, atm_currencies, atm_daily_limit, atm_contactless
  — Для "exchange_point" включай поле: exchange_currencies
  — Для "terminal" включай поле: terminal_services
  — Не включай поля других типов в объект (например, не добавляй atm_type к branch)
  — Если на сайте есть карта с отметками — используй координаты с карты (latitude/longitude)
  — Если у банка десятки банкоматов — собери ВСЕ, не пропускай
  — Не придумывай адреса. Если адреса нет — пиши null в address
  — location_type СТРОГО одно из: branch / atm / exchange_point / terminal / mini_office / cash_desk
  — region СТРОГО одно из: dushanbe / sogd / khatlon / gbao / rrs
  — Верни только JSON-массив, без объяснений и комментариев
