# SEO-план для Basen.uz

> Цель: вывести сайт на первые позиции в Google и Яндекс по запросам о бассейнах Ташкента.
> Разбито на 4 уровня по приоритету — от быстрых побед до долгосрочной стратегии.

---

## Уровень 1 — Техническая база (неделя 1–2)

*Без этого остальное бесполезно*

### 1.1 Schema.org разметка — самый большой рычаг

Каждый бассейн должен иметь JSON-LD на своей странице:

```json
{
  "@context": "https://schema.org",
  "@type": "SportsActivityLocation",
  "name": "Название бассейна",
  "address": {
    "@type": "PostalAddress",
    "streetAddress": "ул. Примерная, 1",
    "addressLocality": "Ташкент",
    "addressCountry": "UZ"
  },
  "telephone": "+998...",
  "priceRange": "от 50 000 сум",
  "geo": {
    "@type": "GeoCoordinates",
    "latitude": 41.299496,
    "longitude": 69.240073
  },
  "openingHours": "Mo-Su 08:00-21:00",
  "aggregateRating": {
    "@type": "AggregateRating",
    "ratingValue": "4.8",
    "reviewCount": "24"
  }
}
```

Google показывает такие сайты с **rich snippets** — рейтинг и часы работы прямо в выдаче.

- [ ] Добавить JSON-LD компонент на страницу `/catalog/[slug]`
- [ ] Добавить AggregateRating если есть данные рейтинга
- [ ] Добавить Schema для главной страницы (`WebSite` + `SearchAction`)

---

### 1.2 sitemap.xml

Модуль `@nuxtjs/sitemap` уже подключён — нужно настроить приоритеты:

| URL | Priority | Changefreq |
|-----|----------|------------|
| `/` | 1.0 | weekly |
| `/catalog` | 0.9 | daily |
| `/catalog/[slug]` | 0.8 | weekly |
| `/map` | 0.7 | weekly |
| `/about`, `/faq` | 0.6 | monthly |
| `/contacts`, `/privacy`, `/terms` | 0.4 | yearly |

- [ ] Настроить приоритеты в `nuxt.config.ts` → `sitemap`
- [ ] Проверить что все страницы бассейнов попадают в sitemap
- [ ] Отправить sitemap в Google Search Console и Яндекс.Вебмастер

---

### 1.3 robots.txt

- [ ] Создать `public/robots.txt`
- [ ] Убедиться что нет блокировок нужных страниц
- [ ] Добавить ссылку на sitemap: `Sitemap: https://basen.uz/sitemap.xml`

---

### 1.4 hreflang для i18n

Три языковые версии (ru/uz/en) должны указывать друг на друга — иначе Google считает их дублями:

```html
<link rel="alternate" hreflang="ru" href="https://basen.uz/catalog" />
<link rel="alternate" hreflang="uz" href="https://basen.uz/uz/catalog" />
<link rel="alternate" hreflang="en" href="https://basen.uz/en/catalog" />
<link rel="alternate" hreflang="x-default" href="https://basen.uz/catalog" />
```

- [ ] Настроить hreflang через `@nuxtjs/i18n` (модуль умеет это автоматически)
- [ ] Проверить в Google Search Console что нет ошибок hreflang

---

### 1.5 Open Graph теги

Влияют на CTR при шаринге в соцсетях и мессенджерах.

- [ ] Добавить `og:image` (1200×630px) для главной и каталога
- [ ] Добавить `og:image` для каждого бассейна (первое фото)
- [ ] Добавить `twitter:card` теги

---

### 1.6 Подключить инструменты мониторинга

- [ ] Google Search Console — подтвердить домен basen.uz
- [ ] Яндекс.Вебмастер — подтвердить домен
- [ ] Google Analytics 4 или Яндекс.Метрика — подключить счётчик

## Уровень 2 — Контентные страницы (неделя 2–4)

*Даёт трафик по длинному хвосту*

### 2.1 Страницы районов — огромный потенциал

Люди ищут именно так: **«бассейн Юнусабад цена»**, **«бассейн Чиланзар»**.

Создать отдельные SEO-страницы:
```
/district/yunusabad   →  «Бассейны Юнусабадского района»
/district/chilanzar   →  «Бассейны Чиланзара»
/district/mirzo-ulugbek → «Бассейны Мирзо-Улугбекского района»
... (все 11 районов)
```

Каждая страница:
- H1: «Бассейны [Район]а Ташкента — цены {год}»
- Список бассейнов этого района с карточками
- Мини-карта района
- Schema.org `ItemList`

- [ ] Создать страницу `/district/[slug].vue`
- [ ] Добавить маршруты для всех районов
- [ ] Прелинковать из каталога и карточек бассейнов

---

### 2.2 Оптимизация страниц категорий

Страницы `/category/[slug]` уже есть. Улучшить:

- [ ] H1 с ключевым словом: «Детские бассейны Ташкента — {count} объектов»
- [ ] Meta description с ценами: «Детские бассейны Ташкента от 30 000 сум. Расписание и фото.»
- [ ] Schema.org `ItemList` со списком бассейнов
- [ ] Текстовый блок 100–200 слов с описанием категории (для индексации)

---

### 2.3 Оптимизация карточек бассейнов `/catalog/[slug]`

- [ ] Title: `{Название} — цены 2026, расписание, фото | Basen.uz`
- [ ] H1 с районом: «Бассейн Дельфин в Юнусабадском районе»
- [ ] Alt-теги на все фото: `{Название бассейна} — фото {номер}`
- [ ] Breadcrumbs: Главная → Каталог → {Категория} → {Название}
- [ ] Schema.org breadcrumb разметка

---

### 2.4 Внутренняя перелинковка

- [ ] С главной → на популярные бассейны и районы
- [ ] С каталога → на страницы районов
- [ ] С карточки бассейна → «Похожие бассейны» (уже есть, усилить)
- [ ] С FAQ → на конкретные страницы каталога
- [ ] Footer: добавить ссылки на популярные районы

## Быстрый старт — топ-4 задачи

1. **Schema.org на страницах бассейнов** — 1–2 дня, даёт rich snippets в выдаче
2. **Google Search Console + Яндекс.Вебмастер** — 30 минут, начать мониторить индексацию
3. **hreflang для ru/uz/en** — 1 день, убирает дубликаты контента
4. **Страницы районов** — самый быстрый рост трафика по длинному хвосту
