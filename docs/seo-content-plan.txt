ПЛАН УЛУЧШЕНИЯ SEO-КОНТЕНТА BASEN.UZ
==================================================

Техническая часть SEO уже сделана (docs/stage-08-seo.md): meta-теги,
schema.org, sitemap, hreflang, GA4. Этот план — про КОНТЕНТ, которого
сейчас не хватает. Без уникального текста на страницах Google не будет
их поднимать выше pools.uz, даже с идеальной техникой.


==================================================
1. АУДИТ: ГДЕ СЕЙЧАС ТОНКИЙ КОНТЕНТ
==================================================

Страницы района `/district/[slug].vue` — САМАЯ БОЛЬШАЯ ДЫРА
  - 11 районов Ташкента, у каждого своя SEO-страница (индексируется,
    есть ItemList/BreadcrumbList schema)
  - Но текста НЕТ ВООБЩЕ — только заголовок + сетка карточек бассейнов
  - Для Google это 11 почти одинаковых "пустых" страниц — риск того,
    что их сочтут дублирующим/тонким контентом и не будут ранжировать
  - Это самый быстрый и дешёвый апгрейд: 11 текстов по 150-250 слов

Страницы категорий `/category/[slug].vue` — текст ЕСТЬ, но:
  - `seoTextMap` в pages/category/[slug].vue:85-92 захардкожен только
    на русском — на /uz/category/* и /en/category/* показывается тот
    же русский текст, что для узбекской/английской версии — минус для
    SEO на этих локалях
  - Текст короткий (2-3 предложения), нет упоминания конкретных
    районов/цен/сезонности — можно расширить до 150-200 слов

FAQ `/faq.vue` — есть FAQPage schema, но:
  - Нужно проверить, покрывает ли реальные вопросы из поиска
    ("где поплавать в Ташкенте", "бассейн рядом", "сколько стоит
    абонемент в бассейн Ташкент") — сейчас не видно ключевых
    гео-вопросов, которые реально ищут

Блога/гида НЕТ вообще
  - Нет раздела для long-tail запросов ("бассейн для ребёнка 3 года
    Ташкент", "куда сходить с детьми летом в Ташкенте", "аквапарки
    на майские в Ташкенте") — это самый дешёвый трафик, но требует
    отдельных страниц-статей, которых сейчас в pages/ нет вообще

Карточки объектов `/catalog/[slug]`
  - Описания на uz/en у многих объектов = null (see data/pools.json,
    напр. monaco, malibu-sun-club) — фолбэк на русский текст на всех
    языковых версиях, обнуляет SEO-пользу мультиязычности


==================================================
2. ПРИОРИТЕТ 1 (2-3 недели) — закрыть дыры на существующих страницах
==================================================

A) Уникальные тексты для всех 11 районов
   - По 150-250 слов на район: какие бассейны есть (открытые/крытые/
     детские), для кого (семьи, спортсмены), метро/транспорт,
     1-2 упоминания конкретных объектов района со ссылкой
   - Хранить как обычный контент-блок на странице (аналогично
     `seoText` в category/[slug].vue), локализовать на uz минимум
     (en можно позже)
   - Пример структуры: "Бассейны Юнусабадского района — от
     фитнес-бассейнов при отелях (Radisson, Wyndham) до открытых
     семейных комплексов. Район хорошо связан метро..."

B) Локализовать `seoTextMap` категорий на uz/en
   - Сейчас: pages/category/[slug].vue, только русский текст
   - Добавить uz/en варианты (можно через locales/*.json как обычные
     ключи вместо хардкода в компоненте — заодно решает и переносимость)
   - Расширить текст: добавить сезонность, диапазон цен, для кого

C) Дозаполнить переводы описаний бассейнов (uz минимум)
   - Приоритет — топ-20 самых посещаемых по GA4 (когда наберётся
     статистика) или пока просто featured: true объекты
   - Не обязательно вручную — можно переводить существующий русский
     текст и я буду ставить готовый uz-перевод при добавлении/правке
     каждого бассейна вперёд

D) Добавить FAQ-вопросы под реальные гео-запросы
   - "Где поплавать в Ташкенте зимой?", "Сколько стоит бассейн в
     Ташкенте?", "Какие бассейны работают круглый год?", "Есть ли
     бесплатные бассейны в Ташкенте?" — брать формулировки из
     Google Search Console (раздел "Запросы") после накопления данных


==================================================
3. ПРИОРИТЕТ 2 (1-3 месяца) — раздел "Гид" (блог)
==================================================

Новый раздел `/guide/[slug]` — статьи-подборки, закрывают long-tail
запросы, которые ни каталог, ни категория, ни район не покрывают.

Формат каждой статьи: intro (100 слов) → список 5-8 бассейнов с
карточками (переиспользовать PoolCard) → вывод/совет → внутренние
ссылки на /catalog, /category, /district.

Первые 10 статей (по убыванию приоритета):
  1. Лучшие бассейны Ташкента 2026 — общий рейтинговый гайд
  2. Детские бассейны Ташкента: куда сходить с ребёнком
  3. Бассейны с подогревом — где плавать зимой в Ташкенте
  4. Аквапарки Ташкента — обзор и цены
  5. Бассейны при отелях Ташкента: можно ли попасть не гостю
  6. Открытые бассейны Ташкента — рейтинг на лето 2026
  7. Спортивные бассейны и дорожки для тренировок в Ташкенте
  8. Бассейны Юнусабада / Чиланзара / Мирзо-Улугбека — топ-подборки
     по самым населённым районам (3 отдельные статьи)
  9. Бассейны с баром и рестораном — летний отдых в Ташкенте
  10. Сколько стоит абонемент в бассейн в Ташкенте — обзор цен

Технически: не нужен CMS/бэкенд — статьи как статические .vue или
.md-страницы через content-модуль, данные бассейнов берутся из
той же pools.json через существующие composables (usePools).


==================================================
4. ПРИОРИТЕТ 3 (постоянно) — внутренняя перелинковка и рост базы
==================================================

- На странице объекта добавить блок "Другие бассейны района X" —
  сейчас PoolSimilar.vue вероятно рекомендует по категории, стоит
  добавить и по району — усиливает district-страницы ссылками
- На district-странице (после блока B выше) — ссылки на релевантные
  guide-статьи ("Топ бассейнов Юнусабада" → сама статья)
  и на 2-3 ближайших района (уже есть блок "другие районы")
- Каждый новый добавленный бассейн — это +1 проиндексированная
  страница с уникальным контентом, так что трек "добавляем по одному"
  (см. parsing-ai/new-pool-template.txt) сам по себе SEO-работа —
  не останавливать параллельно с гайдами


==================================================
5. КАК ИЗМЕРЯТЬ РЕЗУЛЬТАТ
==================================================

- Google Search Console (подключить, если ещё нет) — раздел
  "Эффективность": клики/показы по district- и category-страницам
  до и после добавления текста
- GA4 (уже установлен, G-YBT2XJ63F5) — органический трафик на
  /district/*, /category/*, /guide/* по месяцам
- Цель через 3 месяца: district- и category-страницы получают
  органический трафик наравне с /catalog, а не в разы меньше как
  сейчас (проверить в GA4 текущее распределение перед стартом —
  это и есть базовая точка отсчёта)


==================================================
ПОРЯДОК РАБОТ
==================================================

Неделя 1-2:  тексты для 11 районов (пункт 2A) + локализация
             категорий (2B)
Неделя 3:    расширить FAQ реальными гео-вопросами (2D)
Месяц 2:     запустить /guide с первыми 3-4 статьями (топ из списка
             приоритета 2)
Месяц 3:     ещё 4-6 статей, начать дозаполнять uz-переводы описаний
             бассейнов (2C), подключить внутреннюю перелинковку (4)

Дальше — держать тот же ритм: 1-2 новых бассейна в неделю (рост базы)
+ 1 guide-статья в 2-3 недели, без этого органика не растёт быстрее
конкурентов.
