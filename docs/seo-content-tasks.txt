ЗАДАЧИ ПО docs/seo-content-plan.txt — ПО ПОРЯДКУ ВЫПОЛНЕНИЯ
==================================================
Каждая задача — с описанием что делать, зачем и в каком файле.
Выполняются последовательно, сверху вниз (внутри приоритета можно
делать раньше следующей, если быстрее).


==================================================
ПРИОРИТЕТ 1 — закрыть дыры в существующем контенте (2-3 недели)
==================================================

[x] Задача 1. Написать SEO-тексты для 11 районов Ташкента
    Что: по 150-250 слов на каждый район (Юнусабадский, Чиланзарский,
    Мирзо-Улугбекский, Сергелийский, Шайхонтохурский, Олмазорский,
    Учтепинский, Яшнободский, Мирабадский, Бектемирский, Яккасарайский).
    В тексте — какие бассейны есть в районе (открытые/крытые/детские),
    для кого подходят, транспорт/метро, ссылка на 1-2 конкретных
    объекта района.
    Зачем: сейчас на этих 11 страницах нет вообще никакого уникального
    текста — только заголовок и сетка карточек, риск "тонкого контента".
    Где: новый объект переводов (например locales/ru.json → district.seo_text
    по slug'ам, аналогично категориям) + вывод в pages/district/[slug].vue.

[x] Задача 2. Вставить блок текста на страницы районов
    Что: добавить в pages/district/[slug].vue SEO-текстовый блок
    (аналогично блоку "SEO text block" в pages/category/[slug].vue:38-45),
    подключить тексты из Задачи 1 через computed по slug.
    Зачем: без вывода в шаблон написанные тексты не попадут в HTML
    и не будут работать на индексацию.
    Где: pages/district/[slug].vue.

[x] Задача 3. Локализовать seoTextMap категорий на uz/en
    Что: сейчас в pages/category/[slug].vue:85-92 текст захардкожен
    только на русском в JS-объекте seoTextMap. Нужно перенести тексты
    в locales/ru.json, locales/uz.json, locales/en.json и брать через
    $t по ключу категории, добавить переводы на узбекский (английский
    можно вторым шагом).
    Зачем: на /uz/category/* и /en/category/* сейчас показывается
    русский текст — теряется SEO-польза мультиязычности.
    Где: pages/category/[slug].vue, locales/*.json.

[x] Задача 4. Расширить существующие тексты категорий
    Что: увеличить seoText с 2-3 предложений до 150-200 слов на
    категорию — добавить сезонность, диапазон цен, для кого подходит,
    упоминание конкретных районов, где категория лучше всего
    представлена.
    Зачем: длина и конкретика текста — прямой фактор ранжирования
    по гео-запросам ("открытый бассейн Ташкент" и т.п.).
    Где: тексты из Задачи 3 (locales/*.json).

[x] Задача 5. Собрать реальные гео-вопросы и расширить FAQ
    Что: добавить в pages/faq.vue вопросы под конкретные гео-запросы:
    "Где поплавать в Ташкенте зимой?", "Сколько стоит абонемент в
    бассейн в Ташкенте?", "Какие бассейны работают круглый год?",
    "Есть ли бесплатные бассейны в Ташкенте?" и т.п.
    Зачем: FAQPage schema уже подключена (rich snippet в Google),
    но вопросы должны совпадать с тем, что реально ищут — сейчас
    не факт, что покрыты гео-формулировки.
    Где: pages/faq.vue. Уточнить формулировки лучше после подключения
    Google Search Console (раздел "Запросы") — см. Задачу 12.

ПРИОРИТЕТ 2 — раздел "Гид" (1-3 месяца)
==================================================

[x] Задача 6. Спроектировать структуру раздела /guide
    Что: создать pages/guide/index.vue (список статей) и
    pages/guide/[slug].vue (шаблон статьи: intro → список бассейнов
    через переиспользование PoolCard.vue → вывод → внутренние ссылки
    на /catalog, /category, /district). Данные статей — простой JSON/MD
    файл (например data/guides.json), без CMS и бэкенда.
    Зачем: фундамент для всех статей — без шаблона нельзя публиковать
    контент из списка ниже.
    Где: новые файлы pages/guide/, data/guides.json.

[x] Задача 7. Статья "Лучшие бассейны Ташкента 2026"
    Что: общий рейтинговый гайд, топ-8-10 бассейнов по рейтингу
    из разных категорий.
    Зачем: самый широкий и частотный запрос кластера, приоритет №1.

[x] Задача 8. Статья "Детские бассейны Ташкента: куда сходить с ребёнком"
    Что: подборка детских бассейнов + советы по возрасту/безопасности.

[x] Задача 9. Статья "Где плавать зимой в Ташкенте" (бассейны с подогревом)
    Что: подборка крытых/круглогодичных бассейнов, актуально в
    холодный сезон — сезонный трафик-спайк.

[x] Задача 10. Статья "Аквапарки Ташкента — обзор и цены"
    Что: обзор всех aquapark-объектов каталога с ценами.

[ ] Задача 11. Оставшиеся статьи гайда (по мере роста трафика)
    Что: "Бассейны при отелях: можно ли попасть не гостю",
    "Открытые бассейны — рейтинг на лето", "Спортивные бассейны и
    дорожки для тренировок", "Бассейны Юнусабада/Чиланзара/
    Мирзо-Улугбека — топ по районам" (3 статьи), "Бассейны с баром
    и рестораном", "Сколько стоит абонемент в бассейн — обзор цен".
    Публиковать по 1 статье в 2-3 недели, не все сразу.

ПРИОРИТЕТ 3 — перелинковка и постоянные процессы
==================================================

[ ] Задача 12. Подключить/проверить Google Search Console
    Что: убедиться, что домен подтверждён в GSC (Yandex Webmaster
    уже подключён, см. коммит bc27077 — GSC нужно отдельно), чтобы
    видеть реальные поисковые запросы и текущие позиции.
    Зачем: без GSC весь план делается вслепую — не видно, какие
    формулировки реально приводят трафик, чтобы уточнять Задачи 5, 11.

[x] Задача 13. Блок "другие бассейны района" на детальной странице
    Что: добавить в pages/catalog/[slug].vue/PoolSimilar.vue
    рекомендации не только по категории, но и по району.
    Зачем: усиливает district-страницы внутренними ссылками,
    помогает Google лучше понимать структуру сайта по районам.

[x] Задача 14. Перелинковка район ↔ гайд-статьи
    Что: на district-странице — блок ссылок на релевантные guide-статьи
    ("Топ бассейнов Юнусабада" → сама статья), в статьях — ссылки
    обратно на district-страницы упомянутых районов.
    Зачем: замыкает контентный граф, ускоряет индексацию новых статей.

[ ] Задача 15. Дозаполнять uz-переводы описаний бассейнов
    Что: у части объектов в data/pools.json поле
    translations.uz.description = null (фолбэк на русский). Дозаполнять
    в приоритете featured: true и топ по посещаемости из GA4.
    Зачем: убирает дублирование контента между локалями, даёт
    узбекоязычному поиску собственный уникальный текст.
